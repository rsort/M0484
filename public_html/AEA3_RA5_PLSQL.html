Blocs PL/SQL i variables

Criteri 5.1 · 5.2 · 5.3 — L'estructura bàsica i la declaració de variables

PL/SQL (Procedural Language/SQL) és l'extensió procedimental d'Oracle que permet combinar SQL amb estructures de programació: variables, condicions, bucles i gestió d'errors. Tot codi PL/SQL s'organitza en blocs.

Optional
DECLARE — declaració de variables, cursors, tipus, excepcions
Obligatori
BEGIN — sentències executables (SQL i lògica PL/SQL)
Optional
EXCEPTION — manejadors d'errors
Obligatori
END; — tancament del bloc (seguit de / per executar)
Blocs anònims vs. subprogrames
Un bloc anònim no té nom, s'executa un cop i no es pot tornar a cridar. Els subprogrames (procediments i funcions) tenen nom, es guarden a la BD i es poden cridar moltes vegades. Per executar un bloc al client SQL: acaba amb / en una línia sola.
Exemple 1 · Declaració i ús de variables

      
Explicació

        Les variables es declaren a la secció DECLARE amb el format: nom tipus [:= valor_inicial];

        %TYPE pren automàticament el tipus de la columna referenciada: si la columna canvia de tipus, el codi PL/SQL s'adapta sol.

        %ROWTYPE declara una variable que conté una fila sencera: cada camp és accessible com variable.nom_columna.

        DBMS_OUTPUT.PUT_LINE és el procediment de la biblioteca estàndard per escriure per pantalla. Cal activar-lo amb SET SERVEROUTPUT ON.
      

    


    

      
Exemple 2 · Constants, NOT NULL i tipus booleans

      

      
Explicació

        CONSTANT impedeix que es modifiqui el valor. NOT NULL obliga a inicialitzar la variable. El tipus BOOLEAN és exclusiu de PL/SQL (no existeix a SQL d'Oracle). Pot prendre els valors TRUE, FALSE i NULL.

SQL dins de PL/SQL

        
Criteri 5.4 · 5.5 — SELECT INTO, DML i cursors implícits

      

    

    
PL/SQL permet embeure sentències SQL directament al bloc. La diferència principal és que els SELECT han de retornar exactament una fila usant la clàusula INTO. Per a múltiples files s'han d'usar cursors.


    

      
Atributs del cursor implícit SQL%

      

        Després de qualsevol sentència DML (INSERT, UPDATE, DELETE) o SELECT INTO, PL/SQL emmagatzema informació al cursor implícit:

        SQL%ROWCOUNT — nombre de files afectades · SQL%FOUND — TRUE si n'hi ha ≥ 1 · SQL%NOTFOUND — TRUE si no n'hi ha cap · SQL%ISOPEN — sempre FALSE per a cursors implícits.
      

    


    

      
Exemple 3 · SELECT INTO i DML dins de PL/SQL

      

      
Explicació

        SELECT ... INTO variable recupera exactament una fila. Si no en troba cap llança NO_DATA_FOUND; si en troba més d'una, llança TOO_MANY_ROWS. Les sentències INSERT, UPDATE i DELETE funcionen igual que en SQL pur, però tenim accés als atributs del cursor implícit per saber el resultat.

Estructures de control

        
Criteri 5.7 — IF/ELSIF/ELSE, CASE, LOOP, WHILE, FOR

      

    

    
PL/SQL té les mateixes estructures de control que altres llenguatges procedimentals: condicionals i bucles. Tots els bucles es poden sortir amb EXIT WHEN condició.


    

      
Exemple 4 · Condicionals IF i CASE

      

      
Explicació

        IF ... ELSIF ... ELSE ... END IF — les clàusules ELSIF (sense 'E' final) permeten encadenar condicions. Nota: si la condició és NULL el bloc no s'executa. CASE és l'alternativa més neta per a múltiples valors discrets: existeix en forma simple (compara un valor) i en forma cercada (avalua condicions independents).
      

    


    

      
Exemple 5 · Bucles: LOOP, WHILE i FOR

      

      
Explicació

        LOOP ... EXIT WHEN ... END LOOP — bucle bàsic, surt quan es compleix la condició. WHILE condicio LOOP — avalua la condició abans de cada iteració. FOR i IN 1..n LOOP — la variable de control (aquí i) es declara implícitament, val de 1 a n i s'incrementa sol. FOR i IN REVERSE va de n a 1. CONTINUE salta a la propera iteració.

Cursors explícits

        
Criteri 5.9 — Processar conjunts de files fila a fila

      

    

    
Quan una consulta pot retornar múltiples files, cal un cursor explícit. El programador controla manualment el seu cicle de vida: declarar, obrir, llegir i tancar.


    

      

        
PasSentènciaDescripció

        
          
1. DeclararCURSOR nom IS select;Defineix el nom i la consulta associada. Es fa a la secció DECLARE.

          
2. ObrirOPEN nom;Executa la consulta i posiciona el punter abans de la primera fila.

          
3. LlegirFETCH nom INTO variables;Llegeix la fila actual i avança el punter a la següent.

          
4. TancarCLOSE nom;Allibera els recursos. Sempre cal tancar el cursor.

        
      

    


    

      
Exemple 6 · Cursor explícit manual i atributs

      

      
Explicació

        %NOTFOUND es fa TRUE quan l'últim FETCH no ha retornat cap fila. S'usa per sortir del bucle. %ROWCOUNT compta les files llegides fins al moment. %ISOPEN és TRUE entre OPEN i CLOSE. El patró estàndard és: obrir, llegir en bucle fins a %NOTFOUND, tancar.
      

    


    

      
Exemple 7 · Cursor FOR LOOP (forma simplificada)

      

      
Explicació

        El FOR registro IN cursor LOOP és la forma recomanada: gestiona automàticament el OPEN, FETCH i CLOSE. La variable registro és implícitament de tipus %ROWTYPE del cursor. Els cursors amb paràmetres permeten variar la consulta en cada execució passant valors a l'OPEN.

Tipus de dades composts

        
Criteri 5.4 — RECORD i col·leccions INDEX BY

      

    

    
PL/SQL ofereix tipus de dades que poden contenir múltiples valors: els RECORD (similars als registres d'una taula) i les col·leccions (similars als arrays).


    

      
Exemple 8 · RECORD i matrices associatives (INDEX BY)

      

      
Explicació

        Un RECORD definit per l'usuari agrupa camps de tipus heterogenis: ideal per representar una fila de dades sense usar %ROWTYPE. Una matriu associativa (TABLE OF ... INDEX BY) és com un diccionari: accés per clau numèrica o de text. Mètodes útils: .COUNT, .FIRST, .LAST, .EXISTS(i), .DELETE(i).

Gestió d'excepcions

        
Criteri 5.10 — Capturar i tractar errors en temps d'execució

      

    

    
Una excepció és un error o condició anormal que interromp l'execució normal. PL/SQL permet capturar-les i actuar en conseqüència sense que el programa acabi abruptament.


    

      

        
Excepció predefinidaQuan es llança

        
          
NO_DATA_FOUNDSELECT INTO no retorna cap fila

          
TOO_MANY_ROWSSELECT INTO retorna més d'una fila

          
ZERO_DIVIDEDivisió per zero

          
DUP_VAL_ON_INDEXViolació de restricció UNIQUE o PK

          
VALUE_ERRORError de conversió o truncament de variable

          
OTHERSCaptura qualsevol excepció no tractada específicament

        
      

    


    

      
Exemple 9 · Captura d'excepcions predefinides i pròpies

      

      
Explicació

        La secció EXCEPTION conté un o més manejadors WHEN excepció THEN. WHEN OTHERS THEN ha d'anar sempre al final i captura tot el que no s'ha tractat. SQLERRM retorna el missatge d'error i SQLCODE el número. Les excepcions d'usuari es declaren com a variable de tipus EXCEPTION i es llancen manualment amb RAISE. PRAGMA EXCEPTION_INIT associa una excepció de l'usuari a un codi d'error d'Oracle.

Procediments i funcions

        
Criteri 5.6 — Subprogrames emmagatzemats a la base de dades

      

    

    
Els subprogrames (procediments i funcions) són blocs PL/SQL amb nom, emmagatzemats a la BD i reutilitzables. La diferència fonamental: les funcions retornen un valor, els procediments no (o el fan via paràmetres OUT).


    

      
PROCEDURE
Executa una acció. Pot tenir paràmetres IN (entrada), OUT (sortida) i IN OUT (entrada i sortida). Es crida amb EXECUTE o des d'un bloc PL/SQL.

      
FUNCTION
Calcula i retorna un valor (clàusula RETURN obligatòria). Pot cridar-se dins d'una expressió SQL o PL/SQL. Ha d'evitar efectes secundaris si s'usa en SQL.

    


    

      
Exemple 10 · Procediment amb paràmetres IN, OUT i IN OUT

      

      
Explicació

        IN — el paràmetre entra però no es pot modificar dins el procediment (per defecte). OUT — el procediment hi escriu un valor de retorn; no cal llegir-lo a l'entrada. IN OUT — entra amb un valor i es pot modificar. CREATE OR REPLACE crea o substitueix si ja existia (evita haver de fer DROP primer). Per eliminar: DROP PROCEDURE nom;
      

    


    

      
Exemple 11 · Funció emmagatzemada

      

      
Explicació

        La funció ha de tenir RETURN tipus a la capçalera i almenys un RETURN valor; al cos. Es pot cridar dins un SELECT, en una expressió d'un bloc PL/SQL o com a valor per defecte d'una columna. Si la funció genera efectes secundaris (DML, escritura de fitxers...) no es pot usar directament en SQL sense mesures addicionals.

Paquets (PACKAGE)

        
Criteri 5.6 — Agrupar subprogrames, variables i tipus relacionats

      

    

    
Un paquet agrupa procediments, funcions, variables, cursors, constants i excepcions relacionades en una unitat lògica. Té dues parts: l'especificació (interfície pública) i el cos (implementació privada).


    

      
Avantatges dels paquets

      

        Encapsulament: els detalls interns queden ocults al cos. Rendiment: Oracle carrega el paquet sencer a memòria en la primera crida, millorant l'accés posterior. Reutilització: les variables públiques del paquet persisteixen durant tota la sessió. Dependències: canviar el cos no invalida els objectes que depenen de l'especificació.
      

    


    

      
Exemple 12 · Especificació i cos d'un paquet

      

      
Explicació

        L'especificació declara tot el que és accessible des de fora (públic). El cos implementa els subprogrames i pot declarar elements addicionals privats (variables, funcions auxiliars) que no apareixen a l'especificació. Per cridar un element del paquet: nom_paquet.nom_element. Per eliminar: DROP PACKAGE nom; (elimina especificació i cos). Oracle inclou paquets predefinits útils: DBMS_OUTPUT, DBMS_SQL, UTL_FILE, DBMS_SCHEDULER.

Disparadors (TRIGGER)

        
Criteri 5.8 — Codi que s'executa automàticament davant un event

      

    

    
Un disparador és un bloc PL/SQL associat a un event (INSERT, UPDATE, DELETE sobre una taula, o events DDL/de BD). S'executa automàticament sense que ningú el cridi explícitament.


    

      

        
TipusMomentGranularitat

        
          
BEFOREAbans de l'operació DML. Pot modificar :NEW per canviar les dades.ROW o STATEMENT

          
AFTERDesprés de l'operació DML. Ideal per auditoria.ROW o STATEMENT

          
INSTEAD OFSubstitueix l'operació sobre una vista no actualitzable.ROW

        
      

    


    

      
:NEW i :OLD

      

        Als disparadors de fila (FOR EACH ROW), :NEW.columna conté el valor nou (disponible a INSERT i UPDATE) i :OLD.columna el valor antic (disponible a UPDATE i DELETE). A un disparador de sentència no estan disponibles.
      

    


    

      
Exemple 13 · Trigger BEFORE per validació i auditoria

      

      
Explicació

        El primer trigger valida que el salari nou no sigui inferior al mínim del càrrec: usa RAISE_APPLICATION_ERROR(-20001, missatge) per llançar un error de negoci (codis d'error d'usuari: entre -20000 i -20999). El segon trigger guarda un registre de cada canvi salarial a una taula d'auditoria. Per deshabilitar/habilitar: ALTER TRIGGER nom DISABLE / ENABLE; Per eliminar: DROP TRIGGER nom;
      

    


    

      
Exemple 14 · Trigger INSTEAD OF sobre vista

      

      
Explicació

        Les vistes que combinen múltiples taules (o que usen GROUP BY, DISTINCT, funcions d'agregació...) no es poden actualitzar directament. Un trigger INSTEAD OF substitueix la operació DML intentada sobre la vista i distribueix els canvis a les taules base correctes.
