Manipulació de dades (DML)

Criteri 4.1 · 4.2 · 4.3 — INSERT, UPDATE, DELETE

El DML (Data Manipulation Language) modifica el contingut de les taules. A diferència del DDL, les sentències DML no fan COMMIT automàtic: els canvis es poden desfer amb ROLLBACK fins que s'executi COMMIT o una sentència DDL.

Exemple 6 · INSERT: diverses formes
Explicació
INSERT INTO taula (col1, col2, ...) VALUES (...) — forma explícita i recomanada.
Si no especifiques les columnes, has de proporcionar valors per a totes les columnes en l'ordre de la definició.
INSERT INTO taula SELECT ... — insereix múltiples files a partir d'una consulta, molt eficient per carregar dades massivament.

    
Exemple 7 · UPDATE: actualitzar files
Explicació
Atenció! Un UPDATE sense WHERE actualitza totes les files de la taula. Sempre verifica el WHERE amb un SELECT previ. L'UPDATE pot usar una subconsulta per calcular el nou valor a partir d'una altra taula.

    
Exemple 8 · DELETE i TRUNCATE
Explicació
DELETE FROM taula WHERE condició elimina files específiques (es pot fer ROLLBACK).
TRUNCATE TABLE taula elimina totes les files de forma ràpida, no es pot desfer (és DDL implícit). TRUNCATE és molt més ràpid que DELETE per buidar taules grans.

Transaccions
Criteri 4.5 · 4.6 · 4.7 · 4.8 — COMMIT, ROLLBACK, SAVEPOINT

    
Una transacció és un conjunt de sentències DML que s'han d'executar com una unitat atòmica: o totes s'apliquen o cap. Garanteix la consistència de la base de dades en cas d'error o operació concurrent.

    

      
BEGIN (implícit)

      →
      
INSERT / UPDATE / DELETE

      →
      
SAVEPOINT sp1

      →
      
Més DML...

      →
      
COMMIT

      o
      
ROLLBACK

    

    
Propietats ACID
Atomicitat: tot o res. Consistència: l'estat queda consistent després. Isolament: les transaccions no es veuen entre elles fins al COMMIT. Durabilitat: un COMMIT persiste fins i tot si el sistema falla.

    
Exemple 9 · COMMIT, ROLLBACK i SAVEPOINT
Explicació
SAVEPOINT nom marca un punt dins la transacció al qual podem tornar parcialment.
ROLLBACK TO SAVEPOINT nom desfà els canvis fins al savepoint però no acaba la transacció: cal un COMMIT o ROLLBACK final.
Les sentències DDL fan COMMIT implícit, finalitzant qualsevol transacció oberta.

    
Exemple 10 · Bloqueig i concurrència
Explicació
Oracle implementa la consistència de lectura: una consulta veu un instantani consistent de les dades en el moment que s'inicia. SELECT ... FOR UPDATE bloqueja les files seleccionades per a actualització. NOWAIT retorna error immediatament si les files ja estan bloquejades.
