Gestió d'usuaris i privilegis

Criteri 2.7 · 2.8 — GRANT, REVOKE, perfils i rols

El DCL (Data Control Language) gestiona qui pot fer què a la base de dades. Oracle distingeix entre privilegis de sistema (CREATE TABLE, CREATE USER...) i privilegis d'objecte (SELECT, INSERT, UPDATE sobre una taula concreta).

Privilegis de sistema vs. privilegis d'objecte
Privilegis de sistema: controlen el que un usuari pot fer dins la BD (crear taules, crear usuaris, connectar-se...). Exemple: CREATE SESSION, CREATE TABLE.
Privilegis d'objecte: controlen l'accés a un objecte concret (taula, vista, seqüència...). Exemple: SELECT ON hr.employees, INSERT ON hr.departments.
Exemple 11 · Crear usuaris, atorgar i revocar privilegis
Explicació
GRANT privilegi ON objecte TO usuari atorga permisos. WITH GRANT OPTION permet que l'usuari delegui el privilegi a d'altres. REVOKE retira el privilegi. Els rols agrupen privilegis per facilitar l'administració: en lloc de donar privilegis individualment a cada usuari, es crea un rol i s'assigna. Oracle inclou rols predefinits com CONNECT, RESOURCE i DBA.

Rols predefinits d'Oracle

      

        CONNECT: permet connectar-se i crear taules bàsiques. RESOURCE: inclou els privilegis de CONNECT i permet crear procediments, triggers, etc. DBA: rol d'administrador amb tots els privilegis del sistema. En producció s'aconsella crear rols personalitzats amb només els privilegis necessaris (principi del mínim privilegi).
