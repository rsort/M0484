La sentència SELECT

Criteri 3.1 · 3.2 — Recuperació i filtratge de dades

La sentència SELECT és la base de tot SQL. Permet triar quines columnes veure, de quina taula, aplicar filtres amb WHERE i ordenar amb ORDER BY.

Sintaxi bàsica
SELECT columna1, columna2 FROM taula WHERE condició ORDER BY columna;
Usa SELECT * per treure totes les columnes (útil per explorar, però evita-ho en producció).
Exemple 1 · Seleccionar totes les columnes

      
Explicació
Retorna totes les columnes i files de la taula EMPLOYEES. L'asterisc (*) significa "totes les columnes". Útil per descobrir l'estructura però ineficient en aplicacions reals.

    


    

      
Exemple 2 · Seleccionar columnes concretes amb àlies

      

      
Explicació
Triem només les columnes que necessitem. L'àlies amb AS canvia el nom de la capçalera a la sortida. Si l'àlies té espais cal posar-lo entre cometes dobles: "Nom Complet".

    


    

      
Exemple 3 · Filtratge amb WHERE i operadors

      

      
Explicació
La clàusula WHERE filtra les files. Els operadors disponibles inclouen: =, != o <>, <, >, BETWEEN, IN, LIKE, IS NULL. Les cadenes de text van entre cometes simples i distingeixen majúscules/minúscules.

    


    

      
Exemple 4 · Operadors lògics AND, OR, NOT i ordenació

      

      
Explicació
AND requereix que totes les condicions siguin certes. OR n'és suficient amb una. ORDER BY ordena per defecte de forma ascendent (ASC); afegeix DESC per invertir-ho. Pots ordenar per múltiples columnes separant-les per comes.

    


    

      
Exemple 5 · BETWEEN, IN, LIKE, IS NULL

      

      
Explicació
BETWEEN a AND b equival a >= a AND <= b (inclusiu). IN (v1,v2,...) comprova si el valor és a la llista. LIKE usa comodins: % (zero o més caràcters), _ (exactament un). IS NULL comprova si una columna no té valor (no usar = NULL!).

    


    

      
Exemple 6 · Expressions aritmètiques i DISTINCT

      

      
Explicació
Podem fer càlculs directament al SELECT. DISTINCT elimina les files duplicades de la sortida. El NVL(valor, substitut) reemplaça els NULL per un altre valor per a evitar errors en càlculs.

Funcions d'una sola fila

        
Criteri 3.2 — Transformar i formatar valors

      

    

    
Les funcions d'una sola fila operen sobre cada fila individualment i retornen un resultat per fila. S'agrupen en funcions de caràcter, numèriques, de data i de conversió.


    

      
Exemple 7 · Funcions de caràcter

      

      
Explicació
UPPER/LOWER canvien a majúscules/minúscules. INITCAP posa majúscula a la primera lletra de cada paraula. CONCAT uneix cadenes (equivalent a ||). SUBSTR(cad, inici, longitud) extreu una subcadena. LENGTH retorna la longitud. TRIM elimina espais dels extrems.

    


    

      
Exemple 8 · Funcions numèriques i de data

      

      
Explicació
ROUND(n, decimals) arrodoneix. TRUNC(n, decimals) trunca sense arrodonir. MOD(n, m) retorna el residu de la divisió. SYSDATE retorna la data i hora actuals. MONTHS_BETWEEN calcula mesos entre dues dates. TO_DATE converteix un text en data.

    


    

      
Exemple 9 · Funcions de conversió i condicionals

      

      
Explicació
TO_CHAR(data, 'format') converteix dates a text amb el format indicat. CASE WHEN ... THEN ... ELSE ... END és l'equivalent SQL de l'estructura condicional. NVL2(expr, si_no_null, si_null) tria entre dos valors segons si l'expressió és NULL.

Funcions de grup i agrupaments

        
Criteri 3.5 — Agregar i resumir dades

      

    

    
Les funcions de grup (COUNT, SUM, AVG, MAX, MIN) operen sobre conjunts de files i retornen un sol valor. Amb GROUP BY podem calcular agregats per a cada grup de dades.


    

      
Regla important

      
Tota columna que aparegui al SELECT i no estigui dins d'una funció de grup, ha d'aparèixer al GROUP BY. En cas contrari, Oracle donarà error.

    


    

      
Exemple 10 · Funcions d'agregació bàsiques

      

      
Explicació
COUNT(*) compta totes les files incloent NULLs. COUNT(columna) ignora els NULLs. Les funcions SUM, AVG, MAX, MIN també ignoren els NULLs automàticament. Útil per obtenir estadístiques ràpides de tota la taula.

    


    

      
Exemple 11 · GROUP BY i filtratge de grups amb HAVING

      

      
Explicació
GROUP BY agrupa les files amb el mateix valor en la columna indicada. HAVING filtra els grups (equivalent de WHERE però per a grups). L'ordre d'execució és: WHERE → GROUP BY → HAVING → ORDER BY.

Unions entre taules (JOINs)

        
Criteri 3.3 · 3.4 — Combinar informació de diverses taules

      

    

    
Quan la informació que necessitem es troba repartida en diverses taules, hem d'unir-les. Oracle SQL segueix l'estàndard SQL:1999 amb les clàusules JOIN ... ON, JOIN ... USING i NATURAL JOIN.


    

      
INNER JOIN
Retorna només les files que tenen coincidència en les dues taules. Les files sense parella queden fora.

      
LEFT OUTER JOIN
Retorna totes les files de la taula esquerra, amb NULL on no hi ha coincidència a la dreta.

      
RIGHT OUTER JOIN
Retorna totes les files de la taula dreta, amb NULL on no hi ha coincidència a l'esquerra.

      
FULL OUTER JOIN
Retorna totes les files de les dues taules, amb NULL on no hi ha coincidència en cap dels dos costats.

    


    

      
Exemple 12 · INNER JOIN amb ON i USING

      

      
Explicació
JOIN ... ON permet especificar qualsevol condició d'unió. JOIN ... USING(col) s'usa quan la columna d'unió té el mateix nom a les dues taules i evita l'ambigüitat. En Oracle, quan uses USING, no pots qualificar la columna d'unió amb l'àlies de taula.

    


    

      
Exemple 13 · JOIN de 3 taules i autounió

      

      
Explicació
Podem encadenar tants JOINs com calgui. L'autounió (self-join) uneix la taula amb ella mateixa: imprescindible per a relacions jeràrquiques com la columna MANAGER_ID que referencia EMPLOYEE_ID de la mateixa taula. Cada instància de la taula necessita un àlies diferent.

    


    

      
Exemple 14 · LEFT OUTER JOIN i FULL OUTER JOIN

      

      
Explicació
LEFT OUTER JOIN inclou els empleats que no tenen departament assignat (DEPARTMENT_ID = NULL). Si usem INNER JOIN aquests empleats desapareixerien del resultat. Útil per detectar dades orfes o incompletes. El FULL OUTER JOIN combina els resultats dels dos OUTER JOINs.

Subconsultes

        
Criteri 3.6 — Consultes dins de consultes

      

    

    
Una subconsulta és una sentència SELECT dins d'una altra sentència SQL. S'executa primer i el seu resultat s'usa com a valor o conjunt de valors per a la consulta externa.


    

      
Subconsulta d'una fila
Retorna exactament un valor. S'usa amb operadors comparatius: =, !=, <, >.

      
Subconsulta de múltiples files
Retorna diverses files. S'usa amb IN, ANY, ALL, EXISTS.

    


    

      
Exemple 15 · Subconsulta d'una sola fila

      

      
Explicació
La subconsulta interna s'executa primer i retorna el salari de l'empleat 'Abel'. La consulta externa usa aquest valor per trobar tots els empleats amb salari superior. Si la subconsulta retornés més d'una fila, Oracle donaria error amb l'operador =.

    


    

      
Exemple 16 · Subconsulta de múltiples files amb IN i EXISTS

      

      
Explicació
IN comprova si el valor és a la llista retornada per la subconsulta. EXISTS comprova si la subconsulta retorna alguna fila (més eficient quan la taula interior és gran). NOT EXISTS s'usa per trobar files que no tenen correspondència.

    


    

      
Exemple 17 · Subconsultes amb ANY i ALL

      

      
Explicació
> ANY(subconsulta) significa "major que com a mínim un valor de la subconsulta" (equivalent al mínim). > ALL(subconsulta) significa "major que tots els valors" (equivalent al màxim). = ANY és equivalent a IN.

Operadors de conjunt

        
Criteri 3.7 — UNION, INTERSECT i MINUS

      

    

    
Els operadors de conjunt combinen els resultats de dues o més sentències SELECT. Les dues consultes han de tenir el mateix nombre de columnes amb tipus de dades compatibles.


    

      

        
OperadorDescripcióElimina duplicats?

        
          
UNIONTotes les files d'ambdues consultesSí

          
UNION ALLTotes les files d'ambdues consultesNo

          
INTERSECTNomés les files presents a les dues consultesSí

          
MINUSFiles de la primera que no estan a la segonaSí

        
      

    


    

      
Exemple 18 · UNION, INTERSECT i MINUS

      

      
Explicació
UNION és útil per combinar dades de taules amb la mateixa estructura (per exemple, dades actuals i històriques). INTERSECT troba elements comuns. MINUS troba diferències. L'ORDER BY al final s'aplica al resultat final i ha de fer referència a la posició de la columna o al nom de la primera consulta.

Bones pràctiques i optimització

        
Criteri 3.8 — Escriure consultes eficients

      

    

    
Una consulta correcta i una consulta eficient no sempre és el mateix. Quan les taules creixen, el rendiment pot degradar-se dràsticament si no s'apliquen bones pràctiques.


    

      
✅ Bones pràctiques

      

        Evita SELECT * en producció — tria sempre les columnes que necessites.

        Filtra amb WHERE tan aviat com puguis per reduir el nombre de files processades.

        Usa EXISTS en lloc de IN amb subconsultes grans.

        Afegeix índexs a les columnes usades freqüentment a WHERE i JOIN.

        Evita funcions sobre columnes indexades al WHERE: WHERE UPPER(last_name) = 'KING' no usarà l'índex de last_name.
      

    


    

      
⚠️ Errors freqüents

      

        Usar = NULL en lloc de IS NULL.

        Posar al WHERE una funció d'agregació (AVG, SUM...) — usar HAVING.

        Oblidar que WHERE s'executa abans que GROUP BY.

        Columnes al SELECT no agregades ni al GROUP BY.
