Tipus de dades a Oracle

Criteri 2.1 · 2.3 — Triar el tipus adequat per a cada columna

Triar el tipus de dada correcte és fonamental: afecta l'espai d'emmagatzematge, el rendiment i la integritat de les dades. Oracle té una nomenclatura pròpia que difereix lleugerament de l'estàndard SQL.

Tipus OracleDescripcióExemple d'ús
VARCHAR2(n)Cadena de longitud variable, màx. n caràcters (fins 4000). Recomanat per text.nom, email, descripcions
CHAR(n)Cadena de longitud fixa. Omple amb espais. Útil per codis fixos.codi_país (2), estat (1)
NUMBER(p,s)Número amb p dígits totals i s decimals. NUMBER sense paràmetres = float.salari NUMBER(8,2), id NUMBER(6)
DATEData i hora (dia, mes, any, hora, minuts, segons). No inclou fracció de segon.data_contractació, data_naix
TIMESTAMPCom DATE però amb fraccions de segon i zona horària opcional.timestamp_transacció
CLOBCaràcter Large Object: text de fins a 4 GB.descripcions llargues, articles
BLOBBinary Large Object: dades binàries de fins a 4 GB.imatges, documents, fitxers

Crear taules i restriccions

Criteri 2.2 · 2.4 · 2.5 · 2.8 — CREATE TABLE, ALTER TABLE, DROP TABLE

El DDL (Data Definition Language) defineix l'estructura de la base de dades: crear, modificar i eliminar taules, índexs, vistes i altres objectes. Les sentències DDL fan un COMMIT implícit automàtic.

Restriccions (CONSTRAINTS)
Les restriccions garanteixen la integritat de les dades. Poden definir-se al nivell de columna (inline) o al nivell de taula (out-of-line). Es recomana donar-los un nom descriptiu per facilitar els missatges d'error.
Exemple 1 · CREATE TABLE amb restriccions
Explicació
PRIMARY KEY identifica de forma única cada fila i implica NOT NULL + UNIQUE.
NOT NULL impedeix valors buits.
UNIQUE garanteix que no hi ha valors repetits (permet un NULL).
CHECK valida que el valor compleixi una condició.
FOREIGN KEY ... REFERENCES estableix la integritat referencial.
DEFAULT estableix un valor quan no s'especifica cap a l'INSERT.

    
Exemple 2 · ALTER TABLE: afegir, modificar i eliminar
Explicació
ALTER TABLE ADD afegeix nous elements.
ALTER TABLE MODIFY canvia el tipus o les propietats d'una columna existent.
ALTER TABLE DROP COLUMN elimina una columna (irreversible).
DROP TABLE nom PURGE elimina la taula sense passar per la paperera de reciclatge d'Oracle.

    
Exemple 3 · CREATE TABLE AS SELECT (CTAS)
Explicació
CREATE TABLE ... AS SELECT crea una nova taula a partir del resultat d'una consulta. Copia les dades i l'estructura de columnes però no copia les restriccions (PRIMARY KEY, FOREIGN KEY, etc.). Molt útil per crear còpies de seguretat o taules temporals.

Vistes, seqüències i índexos
Criteri 2.6 · 2.8 — Objectes addicionals de l'esquema

    
A més de les taules, Oracle permet crear altres objectes d'esquema: les vistes simplifiquen l'accés a dades complexes, les seqüències generen valors únics automàtics i els índexos acceleren les consultes.

    
Exemple 4 · Crear i usar vistes
Explicació
Una vista és una consulta emmagatzemada que s'usa com si fos una taula. No emmagatzema dades, sinó la definició de la consulta. CREATE OR REPLACE actualitza la vista si ja existeix. WITH CHECK OPTION impedeix modificar files que quedarien fora de la vista. WITH READ ONLY impedeix qualsevol DML a través de la vista.

    
Exemple 5 · Seqüències per generar IDs únics
Explicació
NEXTVAL retorna el proper valor de la seqüència i l'incrementa (irreversible). CURRVAL retorna el darrer valor obtingut per la sessió actual. En Oracle 12c+ es pot usar la sintaxi columna NUMBER GENERATED ALWAYS AS IDENTITY.
