Tipus de dades a Oracle

Criteri 2.1 · 2.3 — Triar el tipus adequat per a cada columna

Triar el tipus de dada correcte és fonamental: afecta l'espai d'emmagatzematge, el rendiment i la integritat de les dades. Oracle té una nomenclatura pròpia que difereix lleugerament de l'estàndard SQL.

Tipus OracleDescripcióExemple d'ús
VARCHAR2(n)Cadena de longitud variable, màx. n caràcters (fins 4000). Recomanat per text.nom, email, descripcions
CHAR(n)Cadena de longitud fixa. Omple amb espais. Útil per codis fixos.codi_país (2), estat (1)
NUMBER(p,s)Número amb p dígits totals i s decimals. NUMBER sense paràmetres = float.salari NUMBER(8,2), id NUMBER(6)
DATEData i hora (dia, mes, any, hora, minuts, segons). No inclou fracció de segon.data_contractació, data_naix
TIMESTAMPCom DATE però amb fraccions de segon i zona horària opcional.timestamp_transacció
CLOBCaràcter Large Object: text de fins a 4 GB.descripcions llargues, articles
BLOBBinary Large Object: dades binàries de fins a 4 GB.imatges, documents, fitxers

Crear taules i restriccions (DDL)

Criteri 2.2 · 2.4 · 2.5 · 2.8 — CREATE TABLE, ALTER TABLE, DROP TABLE

El DDL (Data Definition Language) defineix l'estructura de la base de dades: crear, modificar i eliminar taules, índexs, vistes i altres objectes. Les sentències DDL fan un COMMIT implícit automàtic.

Restriccions (CONSTRAINTS)
Les restriccions garanteixen la integritat de les dades. Poden definir-se al nivell de columna (inline) o al nivell de taula (out-of-line). Es recomana donar-los un nom descriptiu per facilitar els missatges d'error.
Exemple 1 · CREATE TABLE amb restriccions

      
Explicació

        PRIMARY KEY identifica de forma única cada fila i implica NOT NULL + UNIQUE.

        NOT NULL impedeix valors buits a la columna.

        UNIQUE garanteix que no hi ha valors repetits (permet un NULL).

        CHECK valida que el valor compleixi una condició.

        FOREIGN KEY ... REFERENCES estableix la integritat referencial entre taules.

        DEFAULT estableix un valor quan no s'especifica cap a l'INSERT.
      

    


    

      
Exemple 2 · ALTER TABLE: afegir, modificar i eliminar

      

      
Explicació

        ALTER TABLE ADD afegeix nous elements (columnes o restriccions).

        ALTER TABLE MODIFY canvia el tipus o les propietats d'una columna existent.

        ALTER TABLE DROP COLUMN elimina una columna (irreversible).

        ALTER TABLE RENAME canvia el nom de la taula.

        DROP TABLE nom PURGE elimina la taula sense passar per la paperera de reciclatge d'Oracle.
      

    


    

      
Exemple 3 · CREATE TABLE AS SELECT (CTAS)

      

      
Explicació

        CREATE TABLE ... AS SELECT crea una nova taula a partir del resultat d'una consulta. Copia les dades i l'estructura de columnes però no copia les restriccions (PRIMARY KEY, FOREIGN KEY, etc.). Molt útil per crear còpies de seguretat o taules temporals.

Vistes, seqüències i índexos

        
Criteri 2.6 · 2.8 — Objectes addicionals de l'esquema

      

    

    
A més de les taules, Oracle permet crear altres objectes d'esquema: les vistes simplifiquen l'accés a dades complexes, les seqüències generen valors únics automàtics i els índexos acceleren les consultes.


    

      
Exemple 4 · Crear i usar vistes

      

      
Explicació

        Una vista és una consulta emmagatzemada que s'usa com si fos una taula. No emmagatzema dades, sinó la definició de la consulta. CREATE OR REPLACE actualitza la vista si ja existeix. WITH CHECK OPTION impedeix modificar files que quedarien fora de la vista. WITH READ ONLY impedeix qualsevol DML a través de la vista.
      

    


    

      
Exemple 5 · Seqüències per generar IDs únics

      

      
Explicació

        NEXTVAL retorna el proper valor de la seqüència i l'incrementa (irreversible). CURRVAL retorna el darrer valor obtingut per la sessió actual. En Oracle 12c+ es pot usar la sintaxi columna NUMBER GENERATED ALWAYS AS IDENTITY que és l'equivalent al AUTO_INCREMENT d'altres SGBD.

Manipulació de dades (DML)

        
Criteri 4.1 · 4.2 · 4.3 — INSERT, UPDATE, DELETE

      

    

    
El DML (Data Manipulation Language) modifica el contingut de les taules. A diferència del DDL, les sentències DML no fan COMMIT automàtic: els canvis es poden desfer amb ROLLBACK fins que s'executin COMMIT o una sentència DDL.


    

      
Exemple 6 · INSERT: diverses formes

      

      
Explicació

        INSERT INTO taula (col1, col2, ...) VALUES (...) — forma explícita i recomanada: si s'afegeixen columnes a la taula en el futur, el codi continua funcionant.

        Si no especifiques les columnes, has de proporcionar valors per a totes les columnes en l'ordre de la definició.

        INSERT INTO taula SELECT ... — insereix múltiples files a partir d'una consulta, molt eficient per carregar dades massivament.
      

    


    

      
Exemple 7 · UPDATE: actualitzar files

      

      
Explicació

        Atenció! Un UPDATE sense WHERE actualitza totes les files de la taula. Sempre verifica el WHERE amb un SELECT previ. Es poden actualitzar diverses columnes en una sola sentència. L'UPDATE pot usar una subconsulta per calcular el nou valor a partir d'una altra taula.
      

    


    

      
Exemple 8 · DELETE i TRUNCATE

      

      
Explicació

        DELETE FROM taula WHERE condició elimina files específiques (es pot fer ROLLBACK).

        DELETE FROM taula (sense WHERE) elimina totes les files però es pot desfer.

        TRUNCATE TABLE taula elimina totes les files de forma ràpida, no es pot desfer (és DDL implícit), i reinicia les seqüències associades. TRUNCATE és molt més ràpid que DELETE per buidar taules grans.

Transaccions

        
Criteri 4.5 · 4.6 · 4.7 · 4.8 — COMMIT, ROLLBACK, SAVEPOINT

      

    

    
Una transacció és un conjunt de sentències DML que s'han d'executar com una unitat atòmica: o totes s'apliquen o cap. Garanteix la consistència de la base de dades en cas d'error o operació concurrent.


    

      
BEGIN (implícit)

      →
      
INSERT / UPDATE / DELETE

      →
      
SAVEPOINT sp1

      →
      
Més DML...

      →
      
COMMIT

      o
      
ROLLBACK

    


    

      
Propietats ACID

      

        Atomicitat: tot o res. Consistència: l'estat queda consistent després. Isolament: les transaccions no es veuen entre elles fins al COMMIT. Durabilitat: un COMMIT persiste fins i tot si el sistema falla.
      

    


    

      
Exemple 9 · COMMIT, ROLLBACK i SAVEPOINT

      

      
Explicació

        SAVEPOINT nom marca un punt dins la transacció al qual podem tornar parcialment.

        ROLLBACK TO SAVEPOINT nom desfà els canvis fins al savepoint però no acaba la transacció: cal un COMMIT o ROLLBACK final.

        Les sentències DDL (CREATE, DROP, ALTER) fan COMMIT implícit, finalitzant qualsevol transacció oberta.

        En Oracle, a diferència d'altres SGBD, cal fer COMMIT explícit: no hi ha autocommit per defecte.
      

    


    

      
Exemple 10 · Bloqueig i concurrència

      

      
Explicació

        Oracle implementa la consistència de lectura: una consulta veu un instantani consistent de les dades en el moment en que s'inicia, fins i tot si una altra sessió fa canvis mentre s'executa. SELECT ... FOR UPDATE bloqueja les files seleccionades per a actualització, impedint que altres sessions les modifiquin. NOWAIT retorna error immediatament si les files ja estan bloquejades.

Gestió d'usuaris i privilegis (DCL)

        
Criteri 2.7 · 2.8 — GRANT, REVOKE, perfils i rols

      

    

    
El DCL (Data Control Language) gestiona qui pot fer què a la base de dades. Oracle distingueix entre privilegis de sistema (CREATE TABLE, CREATE USER...) i privilegis d'objecte (SELECT, INSERT, UPDATE sobre una taula concreta).


    

      
Exemple 11 · Crear usuaris, atorgar i revocar privilegis

      

      
Explicació

        GRANT privilegi ON objecte TO usuari atorga permisos. WITH GRANT OPTION permet que l'usuari delegui el privilegi a d'altres. REVOKE retira el privilegi. Els rols agrupan privilegis per facilitar l'administració: en lloc de donar privilegis individualment a cada usuari, es crea un rol i s'assigna. Oracle inclou rols predefinits com CONNECT, RESOURCE i DBA.
