§ 1

Context i necessitat de les BD NoSQL

Per entendre MongoDB cal entendre per què les bases de dades relacionals tradicionals van deixar de ser suficients.

Els grans canvis tecnològics dels darrers 25 anys

L'explosió d'Internet, les xarxes socials i els dispositius mòbils ha generat un creixement exponencial en el volum, la varietat i la velocitat de les dades. Plataformes com Facebook, Instagram o Twitter gestionen centenars de milions d'usuaris generant dades constantment i de manera no estructurada.

Dades que es generen cada minut (2015, en creixement constant)

  • YouTube: 300 hores de vídeo pujades
  • Facebook: 4.166.667 publicacions
  • Twitter: 347.222 tweets enviats
  • WhatsApp: 347.222 fotos compartides
  • Tinder: 590.278 swipes d'usuaris
  • App Store: 51.000 descàrregues d'aplicacions

Avui, la població total d'Internet supera els 5.560 milions de persones (2025), amb una projecció de 6.120 milions el 2026. Cada any es creen aproximadament 300 zettabytes de dades — l'equivalent a 6.000 milions de mòbils de 128 GB completament plens cada dia.

Limitacions de les BD relacionals davant el Big Data

  • Escalabilitat vertical: només creixen afegint maquinari més potent al mateix servidor.
  • Esquema rígid: totes les files d'una taula han de tenir les mateixes columnes.
  • Dificultat amb dades no estructurades: imatges, textos lliures, JSON anidats.
  • Rendiment sota alta concurrència: milions d'usuaris simultanis saturen el sistema.
  • Cost elevat de llicències i infraestructura en solucions tradicionals.

El Big Data: les 5 (o 7) Vs

El Big Data és el sector de IT que fa referència a grans conjunts de dades que, per la velocitat a la qual es generen, la capacitat per tractar-los i els múltiples formats i fonts, és necessari processar-los amb mecanismes diferents als tradicionals. (ONU, 2012)

§ 2

Introducció a les BD NoSQL

Les bases de dades NoSQL (Not Only SQL) són sistemes de gestió de dades que difereixen del model clàssic relacional. Sorgeixen com a resposta a les limitacions d'escala i flexibilitat davant el Big Data.

Característiques principals

  • No utilitzen SQL com a llenguatge de consulta principal.
  • Les dades no requereixen estructures fixes com taules amb columnes predefinides.
  • No garanteixen consistència plena (els canvis es propaguen als servidors amb cert retard).
  • Escalen horitzontalment: augmentant el nombre de servidors (sharding).
  • Generalment open-source i amb menors costos d'infraestructura.

Tipus de BD NoSQL

TipusDescripcióExemples
Clau-ValorEmmagatzema parells clau→valor. Molt ràpid per a lectures simples.Redis, DynamoDB
DocumentalEmmagatzema documents JSON/BSON. Flexible i versàtil.MongoDB, CouchDB
Orientada a ColumnesOrganitza les dades per columnes. Eficient per a consultes analítiques.Cassandra, HBase
GrafsModelitza relacions complexes entre entitats com a nodes i arestes.Neo4j, InfiniteGraph

Segons el rànquing de DB-Engines (maig 2026, 434 sistemes), MongoDB és la 5a base de dades més popular del món i la líder absoluta entre les NoSQL documentals, amb una puntuació de 384,65.

§ 3

MongoDB: la BD documental per excel·lència

Origen i història

  • 2007L'empresa 10gen la desenvolupa mentre construïa una plataforma PaaS (Platform as a Service), similar a Google App Engine.
  • 2009MongoDB és llançat com a producte públic sota llicència de codi obert AGPL.
  • 2011Es llança la versió 1.4, considerada la primera versió llesta per a producció.
  • 2025MongoDB es troba a la versió 8.0.x i és la BD NoSQL amb major popularitat mundial.

Definició

MongoDB és un sistema de base de dades no relacional (NoSQL) open-source orientat a documents. Emmagatzema dades en un format basat en JSON anomenat BSON (Binary JSON), que permet representar estructures de dades complexes de manera natural.

El seu nom prové de la paraula anglesa "humongous" (enormement gran), reflectint la seva capacitat per gestionar grans volums de dades.

Principals avantatges

  • Multi-plataforma: disponible a Windows, macOS i Linux.
  • Alt rendiment: consultes ràpides gràcies a índexs i el format BSON.
  • Alta disponibilitat: suport natiu per a rèpliques (Replica Sets).
  • Molt escalable: sharding automàtic entre múltiples servidors.
  • Esquema flexible: cada document pot tenir una estructura diferent.
  • Integració natural amb JavaScript i entorns web moderns.

Quan utilitzar MongoDB?

  • L'aplicació tindrà un creixement ràpid i necessita escalar horitzontalment.
  • Es treballa amb servidors al núvol (cloud) i es necessita distribució geogràfica.
  • Es vol muntar una base de dades de manera àgil i sense esquema rígid previ.
  • Les dades no tindran sempre la mateixa estructura (perfils d'usuari, catàlegs, logs).
  • La informació és molt dinàmica i canvia freqüentment en la seva forma.
  • Molts usuaris accediran al sistema simultàniament (xarxes socials, e-commerce).

Documents BSON

Un document és la unitat bàsica d'emmagatzematge. És un conjunt de parells clau-valor que representa una entitat del món real. Els valors poden ser String, Number, Array, Date, Boolean o Object.

// Exemple de document d'un usuari de JoinApp
{
  "_id":        ObjectId("64a7f3b2c8d9e0f1a2b3c4d5"),
  "nom":        "Anna García",
  "user":       "anna_garcia",
  "edat":       28,
  "hobby":      "senderisme",
  "correu":     "anna@joinapp.cat",
  "estat_civil":"soltera",
  "fills":      0,
  "data_reg":   ISODate("2024-03-15T10:30:00Z")
}

Col·leccions

Les col·leccions representen un conjunt de documents d'una mateixa entitat. A diferència de les taules SQL, els documents d'una mateixa col·lecció poden tenir camps completament independents entre ells.

Modelat de relacions: documents incrustats

Quan una entitat relacionada s'accedeix sempre conjuntament amb l'entitat principal, la millor opció és incrustar-la dins el mateix document (una sola consulta ho recupera tot).

// Relació 1:1 amb document incrustat
{
  "_id":  "u0001",
  "nom":  "Joan Martí",
  "adreca": {
    "carrer":      "Carrer Major, 15",
    "ciutat":      "Barcelona",
    "codi_postal": "08001"
  }
}

Modelat de relacions: documents referenciats

Quan la relació pot créixer molt (1:N) o quan les dades relacionades s'accedeixen independentment, és millor emmagatzemar una referència (_id) a l'altre document.

// Relació 1:N amb referència — col·lecció 'llibres'
{
  "_id":       987654321,
  "titol":     "MongoDB: The Definitive Guide",
  "autor":     ["K. Chodorow", "M. Dirolf"],
  "pagines":   216,
  "id_editor": "oreilly"   // Referència a col·lecció 'editors'
}
§ 5

Com funciona: instal·lació i ús

Components principals

ComponentDescripció
MongoDB Server (mongod)Motor de la BD. Emmagatzema les dades, gestiona col·leccions i documents, i escolta connexions. S'executa en segon pla com a servei.
MongoDB CompassInterfície gràfica oficial. Permet veure BD, crear col·leccions, inserir i consultar documents i analitzar l'esquema (Schema). Ideal per a principiants.
MongoDB Shell (mongosh)Consola de comandes. Permet escriure instruccions MQL directament. Més potent per a automatitzar tasques i fer consultes complexes.

MongoDB té el seu propi llenguatge de consulta i manipulació d'informació, MQL (MongoDB Query Language), basat en JavaScript amb lleugeres diferències.

Gestió de bases de dades i col·leccions

mongod                         // Iniciar el servidor
mongosh                        // Iniciar el client (Shell)

use joinapp                    // Crear o seleccionar una BD
db                             // Veure la BD actual
show dbs                       // Llistar totes les BD
show collections               // Llistar col·leccions de la BD actual

db.createCollection("usuaris") // Crear col·lecció explícitament
db.productes.drop()            // Eliminar una col·lecció
db.dropDatabase()              // Eliminar la BD actual

Inserció de documents

// Inserir un document (MongoDB genera _id automàticament)
db.usuaris.insertOne({
  "nom":  "Leonardo Kuffo",
  "edat": 21,
  "pais": "Ecuador"
})

// Inserir múltiples documents
db.usuaris.insertMany([
  { "nom": "Anna", "edat": 30 },
  { "nom": "Marc", "edat": 22, "hobby": "escalada" }
])
§ 6

Consultes amb MQL

find() bàsic

db.usuaris.find()              // Tots els documents
db.usuaris.findOne()           // Un sol document
db.usuaris.count()             // Comptar documents

db.usuaris.find({ "nom": "Anna" })          // Cerca exacta
db.usuaris.find().limit(5).skip(2)          // Paginació
db.usuaris.find().sort({ "edat": 1 })       // Ordenar (1=asc, -1=desc)

Operadors de comparació

OperadorSignificatExemple
{ camp: val }Igual a{ "edat": 25 }
$gtMajor que{ "edat": { $gt: 18 } }
$gteMajor o igual{ "edat": { $gte: 18 } }
$ltMenor que{ "preu": { $lt: 50 } }
$lteMenor o igual{ "preu": { $lte: 50 } }
$neDiferent de{ "actiu": { $ne: false } }
$inInclòs en llista{ "rol": { $in: ["admin","user"] } }
$existsCamp existeix{ "telèfon": { $exists: true } }
$regexExpressió regular{ "nom": { $regex: "^A" } }

Operadors lògics

// AND implícit: condicions separades per coma
db.usuaris.find({ "edat": { $gte: 18 }, "pais": "Catalunya" })

// OR amb $or
db.usuaris.find({
  $or: [
    { "hobby": "senderisme" },
    { "hobby": "ciclisme" }
  ]
})

// AND + OR combinats
db.usuaris.find({
  "actiu": true,
  $or: [{ "edat": { $lt: 25 } }, { "fills": 0 }]
})
§ 7

Modificar i eliminar documents

Update total (replaceOne)

Reemplaça completament el document existent per un de nou, mantenint el mateix _id. Els camps no inclosos desapareixen.

db.usuaris.replaceOne(
  { _id: 1 },                   // Filtre
  { nom: "Joan", edat: 30 }     // Reemplaç total
)

Update parcial (updateOne / updateMany)

Modifica únicament els camps especificats, deixant la resta del document intacta.

// Actualitzar un sol document
db.usuaris.updateOne(
  { _id: 1 },
  { $set: { edat: 31 } }
)

// Actualitzar múltiples documents
db.usuaris.updateMany(
  { "pais": "Espanya" },
  { $set: { moneda: "EUR" } }
)

// Incrementar un valor numèric
db.productes.updateOne(
  { "id": "p001" },
  { $inc: { estoc: -1 } }   // Decrementar estoc en 1
)

Operadors d'actualització

OperadorAcció
$setModifica o afegeix un camp.
$unsetElimina un camp del document.
$incIncrementa o decrementa un valor numèric.
$pushAfegeix un element a un array.
$addToSetAfegeix un element a un array, però només si no existeix ja.
$popElimina el primer (−1) o l'últim (1) element d'un array.
$pullElimina elements d'un array que compleixin una condició.

Eliminar documents

// Eliminar el PRIMER document que compleixi la condició
db.productes.deleteOne({ "id": "1" })

// Eliminar TOTS els documents que compleixin la condició
db.productes.deleteMany({ "actiu": false })

// Buidar completament una col·lecció
db.productes.deleteMany({})
